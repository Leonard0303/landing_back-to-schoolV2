# Журнал разработки — KZT (Vite + TypeScript)

Ниже — хронология запросов пользователя и внесённых изменений в рамках создания скелета сайта из 5 блоков: Header, Plan, What_you_get, Tablet, Footer.

---

## 1. Создание скелета из 5 блоков
**Запрос:** сделать основной шаблон сайта — 5 блоков (Header, Plan, What_you_get, Tablet, Footer), каждый в своём TS-файле, равномерно делящих страницу на 5 равных частей.

**Сделано:**
- Созданы компоненты `src/components/Header.ts`, `Plan.ts`, `WhatYouGet.ts`, `Tablet.ts`, `Footer.ts` (каждый экспортирует функцию `createX()`, возвращающую `HTMLElement`).
- `src/main.ts` пересобран: удалён стандартный Vite-шаблон, вместо него импорт и рендер всех 5 компонентов.
- `src/style.css`: `#app` — flex-колонка на весь `100svh`, каждый `.block` — `flex: 1 0 20%` (5 равных секций).

---

## 2. Поля (margins) по бокам
**Запрос:** добавить универсальные боковые поля для приятного вида на ПК, блоки должны быть по центру.

**Сделано:** добавлен `padding-inline: clamp(...)` на `.block`.

**Запрос:** поля только для ПК, на мобильных — без полей.
**Сделано:** добавлена media-query `@media (max-width: 1024px) { padding-inline: 0 }`.

**Проблема:** поля не работали на окне шириной 1024px (совпадало с брейкпоинтом "мобильного" стиля).
**Исправлено:** брейкпоинт сужен до `640px`.

**Запрос:** поля должны быть серыми, видимыми как отдельная зона, а не просто внутренний отступ; блоки должны быть уже и лежать в этой серой зоне на всех устройствах.
**Сделано:** структура переработана — `.block` стал серой полосой на всю ширину (`--margin-bg`), `.block__inner` — узкий центрированный контейнер с цветом конкретной секции. Убрана media-query отключения полей — поля работают всегда.

**Запрос:** уменьшить поля в 2 раза.
**Сделано:** `clamp(16px, 8vw, 120px)` → `clamp(8px, 4vw, 60px)`.

## 5. Блок "Тариф" — 3 карточки планов
**Запрос:** реорганизовать файлы в папку `src/components/Plans/`:
- `Plan_general.ts` — общие настройки/обёртка секции.
- `Plan_1.ts`, `Plan_2.ts`, `Plan_3.ts` — контент каждого тарифа.

Исправить баги после переноса и собрать 3 карточки внутри блока Plan.

**Сделано:**
- Исправлен путь импорта в `main.ts` (`./components/Plans/Plan_general.ts`).
- Каждый `Plan_N.ts` экспортирует `createPlanCard()`.
- `Plan_general.ts` импортирует все 3 карточки и вставляет их в `.plan__grid`.
- Добавлены CSS-стили сетки карточек.

**Запрос:** карточки "висят в воздухе" — сделать их такого же размера, как весь блок Plan.
**Сделано:** `.block__inner` и `.plan__grid` растянуты на всю высоту секции (`align-items: stretch`, `flex: 1`), карточки заполняют доступное пространство.

**Запрос:** заполнить блок планов по референсу (скриншот с тарифами "Всё включено" / "Планшет + SIM" / "Планшет").
**Сделано:** карточки дополнены бейджем номера плана, списком с галочками, местом под иконки, крупной ценой и синей кнопкой "ПОДКЛЮЧИТЬ". Заголовок секции — "Выберите свой план".
